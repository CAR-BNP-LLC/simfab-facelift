# SEO Implementation Plan - SimFab
## Psychology-Driven Technical & Content Strategy

**Last Updated**: 2024  
**Scope**: Complete SEO optimization for full SimFab product ecosystem

---

## Executive Summary

**Goal**: Dominate search results for complete simulation cockpit solutions, mounting hardware, and accessories in flight simulation and racing simulation markets.

**Core Insight**: SimFab doesn't just sell mounts - they sell **complete modular cockpit ecosystems**. Users have different journey stages:
1. **New to sim**: Need complete cockpit system ("flight sim cockpit", "racing cockpit setup")
2. **Upgrading**: Have controller, need mounting ("[Brand] [Model] mount")
3. **Expanding**: Have cockpit, need add-ons ("MFD panel mount", "rudder pedal plate", "monitor stand")
4. **Converting**: Want to switch between racing/flight ("conversion kit", "hybrid cockpit")

**Priority**: Multi-faceted strategy targeting all customer journey stages with emphasis on complete cockpit solutions (highest AOV) and brand+model compatibility (highest conversion intent).

## Part 2: Search Psychology Analysis - Expanded

### 2.1 Customer Journey & Search Intent Mapping

#### **Journey Stage 1: Problem Recognition - "I Want a Complete Setup"** (Commercial Intent)
- *User state*: New to simulation, wants full cockpit
- *Searches*: 
  - `"flight sim cockpit"`, `"complete flight simulator setup"`, `"modular flight sim cockpit"`
  - `"sim racing cockpit"`, `"racing simulator rig"`, `"complete racing setup"`
  - `"best flight sim cockpit"`, `"affordable sim racing cockpit"`
- *Psychology*: High purchase intent, researching complete solutions, price-sensitive but quality-focused
- *AOV**: Highest ($400-$600+ for complete systems)
- *Target pages*: Category pages (`/flight-sim`, `/sim-racing`), Homepage, Product detail for base models

#### **Journey Stage 2: Problem Recognition - "I Have a Controller, Need Mounting"** (Transactional Intent)
- *User state*: Just bought controller (or has one), needs mounting solution
- *Searches*: 
  - `"[Brand] [Model] mount"`, `"[Brand] [Model] desk mount"`, `"[Brand] [Model] cockpit mount"`
  - `"how to mount [brand] [model]"`, `"[brand] [model] compatible mount"`
- *Psychology*: Urgent need, solution-focused, ready to buy, compatibility-critical
- *AOV**: Medium ($100-$300)
- *Target pages*: Product detail pages, Add-on module pages, Compatible brands page

#### **Journey Stage 3: Expansion - "I Have a Cockpit, Need Add-Ons"** (Commercial Intent)
- *User state*: Has base cockpit, wants to upgrade/expand
- *Searches*:
  - `"MFD panel mount"`, `"rudder pedal plate"`, `"monitor stand for sim racing"`
  - `"flight sim add-on modules"`, `"cockpit accessories"`
  - `"keyboard tray for simulator"`, `"armrest for flight sim"`
- *Psychology*: Moderate purchase intent, comparing options, building complete setup over time
- *AOV**: Low-Medium ($50-$200 per add-on)
- *Target pages*: Add-on product pages, Category add-on sections, Assembly manuals (shows compatibility)

#### **Journey Stage 4: Conversion - "I Want to Switch Between Racing/Flight"** (Commercial Intent)
- *User state*: Has one type of cockpit, wants to add other functionality
- *Searches*:
  - `"racing to flight sim conversion kit"`, `"hybrid simulator cockpit"`
  - `"convert racing cockpit to flight sim"`, `"dual purpose sim cockpit"`
- *Psychology*: High purchase intent, wants versatility, considering value
- *AOV**: High (conversion kits often $200-$400)
- *Target pages*: Conversion kit products, Hybrid cockpit pages, Category pages

#### **Journey Stage 5: Comfort & Enhancement** (Lower Intent, Repeat Purchases)
- *User state*: Has working setup, wants comfort/immersion improvements
- *Searches*:
  - `"sim racing seat pillow"`, `"flight sim armrest"`, `"shaker mount for simulator"`
  - `"monitor stand height adjustment"`, `"keyboard tray simulator"`
- *Psychology*: Lower urgency, researching improvements, price-conscious
- *AOV**: Low ($30-$100)
- *Target pages*: Accessory product pages, Category add-on sections

## Part 3: Technical SEO Implementation

### 3.1 Dynamic Meta Tags System

**Problem**: Currently only static meta tags in `index.html`. Product pages can't have unique SEO metadata.

**Solution**: Implement `react-helmet-async` for dynamic per-page meta tags.

#### Implementation Steps:

1. **Install dependency:**
   ```bash
   npm install react-helmet-async
   ```

2. **Create SEO Hook/Component:**
   ```typescript
   // src/hooks/useSEO.ts
   import { Helmet } from 'react-helmet-async';

   interface SEOData {
     title: string;
     description: string;
     keywords?: string[];
     canonical?: string;
     ogImage?: string;
     ogType?: 'website' | 'product';
     noindex?: boolean;
   }

   export const useSEO = (data: SEOData) => {
     return (
       <Helmet>
         <title>{data.title}</title>
         <meta name="description" content={data.description} />
         {data.keywords && <meta name="keywords" content={data.keywords.join(', ')} />}
         {data.canonical && <link rel="canonical" href={data.canonical} />}
         {data.noindex && <meta name="robots" content="noindex, nofollow" />}
         
         {/* Open Graph */}
         <meta property="og:type" content={data.ogType || 'website'} />
         <meta property="og:title" content={data.title} />
         <meta property="og:description" content={data.description} />
         {data.ogImage && <meta property="og:image" content={data.ogImage} />}
         
         {/* Twitter */}
         <meta name="twitter:card" content="summary_large_image" />
         <meta name="twitter:title" content={data.title} />
         <meta name="twitter:description" content={data.description} />
         {data.ogImage && <meta name="twitter:image" content={data.ogImage} />}
       </Helmet>
     );
   };
   ```

### 3.3 Canonical URLs

**Purpose**: Prevent duplicate content penalties when URLs have parameters or multiple paths to same content.

**Implementation:**
- Every page must have `<link rel="canonical" href="...">`
- Canonical should be the cleanest URL (no query params, no trailing slash issues)
- Use absolute URLs: `https://simfab.com/product/{slug}`
- Product pages accessible via ID should canonicalize to slug URL

**Examples:**
- `/product/123` → Canonical: `/product/dcs-flight-sim-modular-cockpit`
- `/product/123?ref=shop` → Canonical: `/product/dcs-flight-sim-modular-cockpit`
- `/flight-sim` → Canonical: `/flight-sim` (no trailing slash)
